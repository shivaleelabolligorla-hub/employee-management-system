# Artifact Inspection

## Objective

Inspect the build artifact generated by Azure DevOps CI Pipeline.

---

Azure DevOps
    ↓
Pipeline Run
    ↓
Artifacts
    ↓
EMS_API

---

## Files Observed

- EMS.API.dll
- EMS.Application.dll
- EMS.Persistence.dll
- EMS.Infrastructure.dll
- EMS.Domain.dll
- appsettings.json
- appsettings.Development.json
- web.config

---

## Understanding

The artifact is a deployment-ready package generated through:

Restore
    ↓
Build
    ↓
Publish
    ↓
Artifact

---

## Difference Between Source Code and Artifact

Source Code:

- Controllers
- Services
- Repositories
- DTOs

Artifact:

- DLL Files
- Configuration Files
- Runtime Assets

---

## Key Learning

Production servers do not require source code.

Only deployment artifacts are required.

---

## Interview Questions

Q: What is contained inside a build artifact?

A:
Compiled assemblies, configuration files, runtime dependencies and deployment assets.

---

Q: What command generates deployment artifacts?

A:
dotnet publish
