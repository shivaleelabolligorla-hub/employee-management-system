# Artifacts And Deployment Flow

## Objective

Understand how software moves from source code to deployed environments.

---

## CI Pipeline Flow

Developer
    ↓
Git Push
    ↓
GitHub
    ↓
Azure DevOps Build
    ↓
Artifact
    ↓
Deployment

---

## Artifact

Definition:

Deployment-ready package generated by build pipeline.

Example:

EMS_API.zip

Contains:

- Application DLLs
- Configuration Files
- Dependencies
- Runtime Assets

---

## Benefits

- Build Once
- Deploy Many Times

Same artifact is deployed to:

- DEV
- QA
- UAT
- PROD

---

## Environment Strategy

DEV

Used by:
Developers

Purpose:
Development Validation

---

QA

Used by:
QA Team

Purpose:
Functional Testing

---

UAT

Used by:
Business Users

Purpose:
User Acceptance Testing

---

PROD

Used by:
End Users

Purpose:
Live Application

---

## IIS Deployment Flow

Artifact
    ↓
Server
    ↓
IIS Website
    ↓

    ↓
Running Application

## Interview Questions

Q: What is an Artifact?

A:
Deployment-ready package generated during CI build.

Q: Why use same artifact for DEV, QA and PROD?

A:
Ensures identical code is validated and promoted through environments.
